# Security Policy

## Supported Versions

Only the [latest release](https://github.com/michaelsanford/IntelcomTracker/releases/latest) is supported with security updates. Older versions will not receive patches.

| Version | Supported |
|---------|-----------|
| Latest  | ✅         |
| Older   | ❌         |

## Reporting a Vulnerability

Please report security vulnerabilities through [GitHub Security Advisories](https://github.com/michaelsanford/IntelcomTracker/security/advisories/new). **Do not open a public issue for security reports.**

### What to include

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if available — reports with proposed fixes will be prioritized)

### Response timeline

- **Acknowledgement and triage:** within 14 days
- **Decision and remediation:** best effort

### Bug bounty

There is no monetary bug bounty. Confirmed reporters will be credited in the README and the documentation site.

## Scope

This policy covers the IntelcomTracker application code and its release artifacts. Third-party dependencies are out of scope but will be addressed if they affect IntelcomTracker users.
